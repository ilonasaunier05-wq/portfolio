05 / Extrait technique

Fonction de transformation

La fonction principale parcourt une matrice et écrit chaque valeur sous forme relationnelle. Cela permet d’unifier des fichiers très larges dans une structure plus adaptée à la base de données. 

mat2rel <- function(dfi, y) {
  m <- as.matrix(dfi)
  lvar <- colnames(dfi)

  for (i in 1:nrow(m)) {
    for (j in 2:ncol(m)) {
      v <- as.character(m[i, j])
      v <- gsub(v, pattern = "\t", replacement = " ")
      v <- gsub(v, fixed = TRUE, pattern = "\\", replacement = "")

      cat(m[i, 1], y, lvar[j], v,
          file = "data_rel.tab",
          sep = "\t",
          append = TRUE)

      cat("\n", file = "data_rel.tab", append = TRUE)
    }
  }
}
06 / Bilan

Un projet utile pour comprendre l’intégration de données

Cube Iris montre une compétence importante : passer de fichiers bruts multiples à une base relationnelle structurée, vérifiable et prête pour l’analyse.

Ce projet m’a permis de travailler la connexion à une base PostgreSQL, la lecture de fichiers volumineux, le nettoyage de données et leur transformation en format relationnel.

Il m’a aussi permis de comprendre l’intérêt d’un cube de données : centraliser plusieurs sources dans une structure commune afin de faciliter les requêtes et l’analyse.
