Détection de sentiment dans les tweets

Ce projet vise à classifier automatiquement des tweets selon leur polarité. Il reprend l’esprit d’un challenge académique autour de la détection de sentiment dans des messages courts, en appliquant un pipeline NLP complet : nettoyage, lemmatisation, vectorisation TF-IDF et comparaison de modèles.

Objectif : transformer des tweets bruts, courts et bruités en représentations exploitables pour prédire automatiquement leur sentiment.

01 / Contexte

Une étude de classification de sentiments

Ce projet porte sur la détection automatique du sentiment dans des tweets. Le sujet m’a été donné dans un cadre pédagogique par mon enseignante, à partir d’un ancien défi académique sur l’analyse de tweets. Le lien du dépôt d’origine est mentionné uniquement comme contexte du sujet.

But de l’étude

L’objectif était de prédire automatiquement la polarité d’un tweet à partir de son contenu textuel. Pour cela, j’ai construit un pipeline complet : nettoyage du texte, traitement des emojis, lemmatisation, vectorisation TF-IDF puis comparaison de plusieurs modèles.

Source du sujet

Le sujet s’inspire d’un ancien travail académique sur la détection de sentiment dans les tweets. Je n’ai pas participé à ce challenge : il a seulement servi de base au sujet transmis en cours.

02 / Pipeline NLP

Du tweet brut à la prédiction

Le projet met en place une chaîne de traitement complète pour passer d’un texte non structuré à une matrice de features utilisable par les modèles.

1

Nettoyage

Suppression des URLs, mentions, hashtags inutiles, ponctuation et caractères parasites.

2

Emojis

Transformation des emojis en tokens interprétables : emoji_laugh, emoji_love, emoji_cry.

3

Lemmatisation

Utilisation de spaCy pour réduire les mots à leur forme canonique.

4

Vectorisation

Création de features TF-IDF avec unigrammes et bigrammes filtrés.

5

Classification

Entraînement de modèles puis génération de fichiers .res pour les prédictions.

04 / Extraits techniques

Les choix importants du code

Deux choix rendent le pipeline plus intelligent : conserver les mots de négation et transformer les emojis en signaux exploitables par le modèle.

Stopwords contrôlés

words_to_keep = {
  "pas", "plus", "jamais", "rien",
  "ne", "non", "trop", "super",
  "bien", "mal", "mdr", "lol"
}

stopwords_spacy = stopwords_spacy - words_to_keep

Emojis utiles

def replace_emojis(text):
    text = emoji.demojize(text, language="en")
    text = text.replace("face_with_tears_of_joy", "emoji_laugh")
    text = text.replace("red_heart", "emoji_love")
    text = text.replace("angry_face", "emoji_angry")
    text = text.replace("crying_face", "emoji_cry")
    return text.replace(":", " ")
05 / Bilan

Un mini projet NLP complet

Ce projet montre une démarche concrète de machine learning appliquée au texte : prétraitement, création de features, comparaison de modèles, évaluation et export des prédictions.

La difficulté principale vient du format tweet : les messages sont courts, parfois ironiques, et contiennent beaucoup d’éléments non standards comme les emojis, les abréviations ou les majuscules.

Le projet montre donc l’importance du prétraitement NLP avant l’entraînement : un simple nettoyage naïf peut supprimer des informations importantes comme la négation ou l’intensité.

Compétences mobilisées
  • Nettoyage de données textuelles
  • Lemmatisation avec spaCy
  • Gestion des stopwords et exceptions
  • Vectorisation TF-IDF
  • Classification supervisée avec scikit-learn
  • Évaluation par accuracy et F1-score macro
