Projet data mining · Classification

Prédiction de l’approbation des prêts

Ce projet utilise des méthodes de classification pour prédire si une demande de prêt doit être acceptée ou refusée à partir de données financières et personnelles.

Objectif : comparer plusieurs modèles et trouver le meilleur compromis entre performance, sécurité bancaire et volume de clients acceptés.

Fiche projet

Type Classification supervisée
Domaine Banque · Crédit · Risque
Données Loan Approval Dataset · 4 270 lignes · 11 features
Cible loan_status : Approved / Not Approved
01 / Objectif

Prédire une décision bancaire

Les banques doivent décider rapidement si elles acceptent ou refusent un prêt. Le projet transforme ce problème métier en tâche de classification, avec une attention particulière portée au F1-score.

Problème

Décision binaire

Prédire une classe : prêt approuvé ou non approuvé, à partir de variables financières et personnelles.

Évaluation

F1-score

Mesure centrale car elle équilibre précision et rappel, donc sécurité et volume de clients acceptés.

Comparaison

Plusieurs modèles

ZeroR, KNN, Naive Bayes, Decision Tree et Random Forest ont été comparés sur DEV puis TEST.

02 / Données

Un dataset léger mais informatif

Les données ont été séparées en 70% train, 15% dev et 15% test afin d’éviter le sur-apprentissage et d’évaluer les modèles sur des données jamais vues.

Split du dataset

Répartition utilisée pour entraîner, optimiser puis tester les modèles.

Train70%
Dev15%
Test15%

Points importants

Volume

4 270 lignes

Le dataset reste suffisamment compact pour comparer rapidement plusieurs approches.

Features

11 variables

Des variables financières et personnelles sont utilisées pour expliquer la décision.

Variable clé

CIBIL Score

Variable principale, corrélée à 77% avec la cible et liée à la capacité de remboursement.

Déséquilibre

Classes inégales

Le jeu de données contient davantage de prêts acceptés que refusés.

03 / Modèles

Comparer les performances

Clique sur une métrique pour visualiser les performances DEV. Le Random Forest obtient le meilleur F1-score sur le corpus DEV, tandis que le Decision Tree optimisé est très compétitif sur le TEST.

Accuracy sur le corpus DEV

Random Forest atteint 98% d’accuracy sur DEV, devant Decision Tree à 97%.

04 / Résultats finaux

Le meilleur modèle dépend du contexte

Sur le corpus TEST, le Decision Tree optimisé obtient le meilleur F1-score. Le Random Forest reste très robuste sur toutes les features initiales.

Focus feature

Le rôle décisif du CIBIL score

L’expérience sans CIBIL score sur KNN entraîne une perte de presque 27% de performance. Cela confirme l’importance de cette variable dans la prédiction.

05 / Bilan

Un projet court mais très parlant

Cette page peut rester plus condensée qu’un projet prioritaire, tout en montrant clairement la démarche, les métriques, le raisonnement et les résultats.

Le projet montre une démarche complète de data mining : compréhension du problème, préparation du split, comparaison de modèles, choix de métriques adaptées, optimisation des features et interprétation des résultats.

L’intérêt principal est d’expliquer pourquoi une bonne performance ne suffit pas : il faut aussi comprendre les variables importantes et le compromis entre précision, rappel et F1-score.
